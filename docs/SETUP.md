# 설치 가이드 (사람용 상세)

> 복붙 방식 기준. clasp 자동화는 [AGENTS.md](../AGENTS.md) 참고.
> 🔴 = 사람이 꼭 해야 하는 단계 / 🟢 = 자동(클릭 한 번에 코드가 처리)

---

## 1. 🔴 양식 가져오기
1. 새 구글시트 생성 ([sheets.new](https://sheets.new))
2. `파일 → 가져오기 → 업로드` → `docs/거래명세서양식.xlsx` 선택
3. 가져오기 위치 **"새 시트 삽입"** → 가져오기
4. → `자료입력` / `세금계산서양식` 두 탭이 생기면 성공

## 2. 🔴 코드 붙여넣기
1. `확장 프로그램 → Apps Script`
2. ⚙ `프로젝트 설정` → **"에디터에 appsscript.json 매니페스트 파일 표시" 체크**
3. 파일 3개:
   - `appsscript.json` ← `gas/appsscript.json`
   - `Code.gs` ← `gas/Code.gs`
   - `Setup.gs` ← (`+` → 스크립트, 이름 `Setup`) `gas/Setup.gs`
4. 💾 저장

## 3. 🟢 설치 실행 (+ 🔴 권한 1회)
1. 시트로 돌아와 새로고침(F5) → 상단 **`📨 거래명세서`** 메뉴 생성
2. `📨 거래명세서 → ⚙ 관리 → 설치 / 갱신` 클릭
3. 🔴 **권한 승인**(최초 1회): `고급 → (안전하지 않음으로) 이동 → 허용`
   - 구글 미인증 개인 스크립트라 뜨는 정상 단계. 본인 시트/메일/드라이브만 사용.
4. 🟢 → `설정·거래처·정기항목·거래내역·발송로그` 시트 생성 + 양식이 한 시트로 통합 + 드롭다운·검증·수식 자동 적용

## 4. 🔴 내 데이터 입력

### 설정 시트
| 키 | 넣을 값 |
|----|---------|
| 공급자_상호 / 사업자번호 / 대표자 / 주소 / 업태 / 종목 | 내 회사 정보 (사업자번호는 하이픈 자동) |
| 공급자_입금계좌 | 입금 계좌 (메일 본문용) |
| 관리자_이메일 | 발송결과 요약 받을 내 메일 |
| 테스트_수신메일 | 테스트용 내 메일 (POC는 전부 여기로만) |
| 발송일 / 발송시각 | 자동발송 날짜(1~31) / 시각(0~23) — 드롭다운 |
| 발송_활성화 | 처음엔 **FALSE** (PDF만, 메일 X) |

### 거래처 시트
- 거래처ID(C001…)·상호·이메일 (필수) / 사업자번호·발송여부 등 선택

### 정기항목 시트
- 매달 청구할 품목·수량·단가 (거래처ID 선택 → 상호 자동표시)

### 세금계산서양식의 은행/계좌 칸
- PDF에 찍히는 계좌. 양식 시트에서 직접 본인 계좌로 수정.

## 5. 테스트 → 실전
1. `👀 미리보기` → Drive `거래명세서_보관` 폴더에서 PDF 확인
2. `⚙ 관리 → 테스트 발송 (나에게 1건)` → 테스트메일 확인
3. 이상 없으면 🔴 `설정`의 **`발송_활성화 = TRUE`**
4. `📧 이번 달 발송하기`(확인창) 또는 `⚙ 관리 → 자동발송 켜기`

## 자주 묻는 문제
| 증상 | 해결 |
|------|------|
| 설치 시 "양식을 먼저 가져오세요" | 1단계(양식 가져오기) 후 다시 |
| "시트 없음: 세금계산서양식" | 양식 탭 이름이 `자료입력`/`세금계산서양식` 인지 확인 / 양식 재가져오기 |
| 권한 화면에서 막힘 | `고급 → 이동 → 허용` |
| `ScriptApp ... 권한` 오류 | `appsscript.json`(스코프) 붙여넣고 재실행 → 권한 재승인 |
| 메일 안 옴 | `발송_활성화=TRUE`? / `테스트_수신메일`·`거래처 이메일` 확인 / 스팸함 / 하루 100통 한도 |
| 자동발송 시각에 안 옴 | `발송일`이 오늘 날짜? + 구글 트리거는 1시간 이내 실행 / 즉시 검증은 `테스트: 1분 후 자동발송` |
| 사업자번호가 밀려 보임 | 10자리 사업자/고유번호인지 확인 (하이픈은 자동) |
| 특정 행 안 보내고 싶음 | `거래내역`의 `발송상태`를 `보류`로 |
