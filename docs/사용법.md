# 사용법 (매월 운영)

> 설치가 끝난 뒤, **매달 실제로 어떻게 보내는지** 정리한 문서입니다. 설치는 [SETUP.md](SETUP.md) 참고.

---

## 🗓 매달 하는 일 (기본 흐름)

1. `정기항목` 시트 확인 — 매달 같은 내용이면 **손댈 것 없음** (1회 등록 후 그대로)
2. 이번 달만 있는 청구 → `거래내역`에 직접 한 줄 추가 (**청구월**을 이번 달로)
3. `📨 거래명세서 → 👀 미리보기` — 발송 없이 PDF만 확인
4. `📧 이번 달 발송하기` — 확인창에서 건수 보고 발송
5. 결과는 `발송로그` 시트 + 관리자 메일로 요약 도착

> **발송 대상** = 이번 달(당월) + `발송상태 = 미발송` + 거래처 `발송여부 = TRUE` 인 건만.

---

## 🔴 "발송하기를 눌렀는데 메일이 안 가고 PDF만 생겼어요"

`발송로그`에 **`생성만(발송보류)`** 로 찍혔다면, **안전장치(발송보류 모드)** 가 켜진 것입니다. 버그가 아닙니다.

- 원인: `설정` 시트의 **`발송_활성화`** 값이 **`FALSE`** → PDF만 만들고 실제 메일은 안 보냄
- 처음 설치 시 실수 발송을 막으려고 기본값이 `FALSE` 입니다.

### 해결
1. `설정` 시트에서 **`발송_활성화`** 값을 드롭다운에서 **`TRUE`** 로 변경
2. 다시 `📧 이번 달 발송하기` → 확인창에 **"실제 메일을 발송합니다"** 로 뜨면 정상
3. 걱정 마세요 — 발송보류로 만든 건은 발송상태가 여전히 `미발송` 이라 **다시 눌러도 중복 발송되지 않습니다.**

> 한 번 `TRUE` 로 바꾸면 계속 유지됩니다. 이후에는 그냥 발송하면 됩니다.

## 📅 다음 달(예: 8월)에 보낼 때 — 지난달 데이터는 어떻게?

**지난달 거래내역은 지우지 마세요. 그대로 두면 됩니다.**

이 시스템은 **`청구월`(예: `2026-08`) 로 발송 대상을 구분**합니다. 오늘 날짜 기준으로 이번 달 청구월을 자동 계산해, **그 달 건만** 골라 보냅니다. 지난달 행은 자동으로 걸러지므로 섞이지 않습니다. (지난 기록으로 남겨두는 게 좋습니다.)

### 다음 달에 실제로 할 일
| 대상 | 할 일 |
|------|-------|
| **정기항목**(매달 같은 거래처) | **아무것도 안 해도 됨** — 발송하기(또는 미리보기) 누르면 이번 달치 거래내역이 자동 생성 (여러 번 눌러도 중복 안 됨) |
| **이번 달만 있는 일회성 청구** | `거래내역`에 직접 한 줄 추가 — 이때 **청구월을 이번 달로** 적어야 포함됨 |
| **발송상태** | 새로 생긴 행은 자동으로 `미발송` — 신경 안 써도 됨 |

---

## ⚙️ `청구월_기준` (당월 / 전월) — 한 번만 확인

`설정` 시트의 **`청구월_기준`** 값에 따라 "이번 달에 누르면 무엇이 나가는지"가 달라집니다.

| 설정값 | 8월 중에 발송하기를 누르면 |
|--------|--------------------------|
| **당월** (기본값) | **8월분**(청구월 2026-08)을 발송 |
| **전월** | **7월분**(청구월 2026-07)을 발송 — 지난달 실적을 다음 달 초에 정산해 보내는 방식 |

- 이번 달치를 이번 달에 보낸다 → **당월**
- 지난달 실적을 다음 달 초에 정산해 보낸다 → **전월**

---

## 🚫 특정 거래처만 이번엔 빼고 싶을 때

- **이번 달만 제외**: `거래내역`에서 그 행의 `발송상태` 를 **`보류`** 로 변경
- **아예 매달 제외**: `거래처` 시트에서 그 거래처의 `발송여부` 를 **`FALSE`** 로

---

## 🤖 매월 자동으로 보내기

매달 직접 누르기 번거로우면:

1. `설정` 시트의 `발송일`(1~31) · `발송시각`(0~23) 지정
2. `📨 거래명세서 → ⚙ 관리 → 자동발송 켜기`
3. 지정 날짜·시각에 자동 발송 (구글 트리거 특성상 **1시간 이내** 실행)

> 즉시 동작 확인은 `⚙ 관리 → 테스트: 1분 후 자동발송` (전부 테스트메일로만 감).
