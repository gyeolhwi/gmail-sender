# 거래명세서 자동 발송 시스템 — 기획 및 요구사항 정의서

- **작성일**: 2026-06-29
- **문서 상태**: 초안 (요구사항 확정 / 일부 항목 사용자 확인 필요)

## 1. 프로젝트 개요

매월 거래처(고객)에게 보내는 **거래명세서를 PDF로 자동 생성하고 Gmail로 발송**하는 자동화 시스템을 구축한다.
현재는 수기로 양식에 입력하고 메일머지(YAMM 추정)로 발송하던 작업을, **Google Sheets + Google Apps Script** 기반으로 자동화한다.

### 1.1 목표
- 거래명세서 작성 → PDF 변환 → 메일 발송까지 **사람 개입 없이 매월 자동 처리**
- 거래처별 **여러 품목**을 한 장의 명세서에 정리
- 발송 결과(성공/실패)를 **로그로 추적**

### 1.2 비목표 (이번 범위 아님)
- 회계/세금계산서 전자발행(국세청 연동) — 대상 아님
- 결제·수금 관리 — 대상 아님
- 웹 UI 별도 구축 — 대상 아님 (구글 시트가 UI 역할)

## 3. 기술 아키텍처

### 3.1 선정: Google Apps Script (GAS)
**Google Sheets + PDF + Gmail** 조합에서 가장 단순하고 비용·인증 부담이 적은 방식.

```
[Google Sheets 워크북]
   ├─ 설정 시트          (공급자 정보 / 발송 설정 / 메일 템플릿)
   ├─ 거래처 시트         (고객 마스터: 상호, 이메일, 사업자정보)
   ├─ 거래내역 시트       (월별 품목 데이터, 거래처별 다중 행)
   ├─ 명세서양식 시트     (PDF로 출력될 거래명세서 레이아웃)
   └─ 발송로그 시트       (발송 결과 자동 기록)
          │
          ▼
[Google Apps Script]
   ① 시간 기반 트리거(매월 N일) 발동
   ② 거래내역에서 해당 청구월·미발송 건을 거래처별 그룹화
   ③ 거래처별로 명세서양식 시트에 데이터 채움
   ④ 양식 시트 영역 → PDF 변환 (Drive 임시 저장)
   ⑤ GmailApp으로 PDF 첨부 발송
   ⑥ 발송로그 기록 + 거래내역 발송상태 갱신
   ⑦ 실패 건 요약을 관리자에게 알림
```

### 3.2 선정 이유
- **무서버**: 별도 호스팅·운영 비용 없음
- **PDF 변환 내장**: 구글 시트 export(`/export?format=pdf`)로 양식 그대로 PDF화
- **Gmail 발송 내장**: `GmailApp.sendEmail()` 한 줄, 추가 SMTP 인증 불필요
- **스케줄 내장**: `ScriptApp.newTrigger()` 시간 기반 트리거로 매월 자동 실행

### 3.3 제약 / 한도
- Gmail 일일 발송 한도: 일반 Gmail 100통/일, Google Workspace 1,500통/일 → **거래처 22곳이라 여유**
- GAS 단일 실행 최대 6분 → 22건 처리에 충분, 향후 수백 건이면 분할 처리 필요
- 첫 실행 시 Drive·Gmail·Sheets 접근 **권한 승인 1회** 필요

### 3.4 비용 — 전 구간 무료 (멤버십 불필요) ✅
사용자 요구사항: **추가 비용(멤버십·구독) 발생 금지**. 선택한 스택은 이 규모에서 **완전 무료**다.

| 구성요소 | 비용 | 근거 |
|----------|------|------|
| Google Apps Script | **무료** | 스크립트 실행 자체에 과금 없음 |
| Google Sheets | **무료** | 개인 구글 계정 기본 제공 |
| Google Drive (PDF 임시 저장) | **무료** | 15GB 무료 용량 공유. PDF 1건 ≈ 50~200KB, 월 22건 → 연간 수 MB 수준으로 미미 |
| Gmail 발송 (GmailApp) | **무료** | 개인 Gmail 기준 Apps Script 이메일 한도 **100통/일** 내 무료. 월 22통이라 여유 |
| 트리거(스케줄) | **무료** | 시간 기반 트리거 무료. 월 1회 실행이라 일일 실행시간 한도와 무관 |

> **중요 — 기존 유료 구독 제거 가능**: `고객정보.xlsx`의 `Merge status`·`Attachment` 컬럼은 **YAMM(Yet Another Mail Merge)** 류 유료 메일머지 애드온의 흔적이다. 본 자동화는 애드온 없이 순수 GAS로 동작하므로, **YAMM 등 기존 멤버십을 해지해 비용을 오히려 줄일 수 있다.**

### 3.5 비용·한도 관련 변수 상황 (모니터링 포인트)
"무료 유지"를 깨뜨릴 수 있는 변수와 대응:

| 변수 | 영향 | 임계점 | 대응 |
|------|------|--------|------|
| **발송 계정 종류 (Q8)** | 개인 Gmail = 100통/일, Workspace = 1,500통/일 | 하루 100통 초과 | 현재 22통/월이라 무관. 단, **하루에 테스트·재발송을 100통 넘게 몰면 그날 차단**(비용 아님, 다음날 자동 해제) |
| **거래처 수 증가** | 발송량 증가 | 개인 100통/일 | 100곳까지는 1회 실행으로 무료 처리. 그 이상이면 **날짜 분할 발송**(여전히 무료). 1,500곳 초과 시에만 Workspace(유료) 검토 |
| **Drive PDF 누적** | 저장용량 잠식 | 15GB | **PDF를 연/월 폴더에 영구 보관**(요청사항). 1건 ≈ 50~200KB, 연 264건이라도 수십 MB → 15GB 무료 용량으로 수십 년치 보관 가능 |
| **GAS 실행시간** | 6분/회 초과 시 중단 | 6분 | 22건은 충분. 대량 시 **배치 분할**(무료) |
| **개인 Gmail 발신 신뢰도** | 스팸 분류 가능(비용 아님) | — | 월 22통 소량이라 위험 낮음. 회사 도메인 발신이 신뢰도↑지만 Workspace 유료 → **무료 유지를 위해 개인 Gmail 사용 권장** |

**결론적 방향성**: 현재 요구사항(거래처 22곳, 월 1회, 22통)은 **개인 무료 Gmail + 순수 GAS**로 100% 무료 운영 가능. 유료 전환은 *거래처 수백 곳 + 빈번 발송* 같은 먼 미래 시나리오에서만 발생하며, 그때도 분할 발송으로 회피 가능하다.

### 4.3 `거래내역` 시트 (월별 품목 데이터)
이번 자동화의 핵심 입력. **거래처당 여러 행**(품목별 1행).

| 컬럼 | 필수 | 설명 |
|------|:---:|------|
| **거래처ID** | ✅ | `거래처` 시트와 매칭 |
| **청구월** | ✅ | 예: 2026-06 (그룹화·발송 기준) |
| 거래일자 | | 양식의 월/일 표기 |
| **품목** | ✅ | 예: 예시품목1, 예시품목2 |
| 규격 | | |
| **수량** | ✅ | |
| **단가** | ✅ | |
| 공급가액 | 자동 | = 수량 × 단가 (수식 또는 스크립트 계산) |
| 세액 | 자동 | = 공급가액 × 10% (※ 부가세 별도 서식) |
| 발행금액 | 자동 | = 공급가액 + 세액 |
| **발송상태** | ✅ | 미발송 / 발송완료 / 실패 (스크립트가 갱신) |

### 4.4 양식 시트 — 기존 파일 그대로 사용 (`자료입력` + `세금계산서양식`)
기존 `거래명세서 양식.xlsx`를 구글 시트로 가져와 **그대로** 쓴다. 양식을 새로 그리지 않는다.
- **`자료입력`**: 스크립트가 공급받는자·품목·작성일을 써넣는 입력 시트
- **`세금계산서양식`**: `자료입력` 값을 **70개 수식(MID 등)으로 끌어와 파란 거래명세서를 자동 렌더링**(등록번호·금액을 자릿수 칸에 자동 분해) → **이 시트를 PDF로 내보냄**

양식이 담는 항목(참고):

- 헤더: "거 래 명 세 서", 보관용/공급받는자용, 책번호·권·호
- 공급자 블록: 등록번호·상호·대표자·주소·업태·종목 (← `설정` 시트에서 고정)
- 공급받는자 블록: 상호·등록번호·대표자·주소·업태·종목 (← `거래처` 시트)
- 작성 년월일 / 공급가액·세액 자릿수 칸
- 품목 표: 월·일·품목·규격·수량·단가·공급가액·세액·비고 (← `거래내역`, 거래처별 다중 행)
- 합계금액 / 입금계좌 / "이 금액을 청구함" 문구

> **품목 행 수**: 기존 양식이 **품목 4행**이라 한 명세서당 **최대 4개**.
> 4개를 초과하면 자동화가 **실패 처리 + 관리자 통지**(임의 누락 방지). 필요하면 양식의 품목 행을 늘려 확장 가능.

## 5. 처리 흐름 (매월 자동 발송)

```
1. 트리거 발동 (매월 {발송일}일)
2. 청구월 결정 ({청구월_기준} = 전월 또는 당월)
3. 거래내역에서 [청구월 일치 AND 발송상태=미발송] 행 조회
4. 거래처ID 기준으로 그룹화
5. 각 거래처 그룹에 대해:
     a. 거래처 마스터에서 공급받는자 정보·이메일 조회
     b. 이메일 없음/발송여부=FALSE → 건너뛰고 로그에 사유 기록
     c. 명세서양식 시트에 공급자·공급받는자·품목·합계 채움
     d. 해당 시트 영역을 PDF로 변환 → 보관 폴더(연/월)에 저장
     e. Gmail로 PDF 첨부 발송 (제목·본문 템플릿 치환)
     f. 성공 시: 거래내역 발송상태=발송완료, 발송로그=성공
        실패 시: 발송상태=실패, 발송로그=실패(오류메시지)
6. 처리 종료 후 관리자에게 결과 요약 메일
     (예: "총 22건 중 성공 21 / 실패 1")
```

### 발송 모드 (안전장치)
`설정.발송_활성화 = FALSE`이면 PDF만 생성하고 발송은 보류 → **첫 도입 시 검토용**으로 권장.

## 7. 메일 발송 사양

### 7.1 첨부 / 파일명
- 형식: PDF
- 파일명 기본 규칙: `{청구월}_거래명세서_{상호}.pdf` (예: `2026-06_거래명세서_예시거래처7.pdf`)
- 거래처 시트 `파일명규칙` 입력 시 우선 적용

**Drive 보관 (아카이브)** — 발송한 PDF는 전부 Drive에 영구 보관:
- 폴더 구조: `거래명세서_보관/{연도}/{청구월}/` (예: `거래명세서_보관/2026/2026-06/`)
- 발송로그 `PDF링크`에 각 파일 링크 기록 → 시트에서 클릭해 바로 열람
- 용량: 1건 ≈ 50~200KB, 연 264건이라도 수십 MB (무료 15GB로 수십 년 보관)
- 발송 성공/실패와 무관하게 생성된 PDF는 보관(재발송·증빙용)

### 7.2 본문 템플릿 (치환변수 예시)
```
{담당자}님 안녕하세요, 예시상사입니다.

{청구월} 거래명세서를 첨부드립니다.
- 공급가액: {공급가액}원
- 부가세: {세액}원
- 합계(청구금액): {발행금액}원

입금계좌: {공급자_입금계좌}

감사합니다.
예시상사 드림
```
> 치환변수: `{담당자}`, `{상호}`, `{청구월}`, `{공급가액}`, `{세액}`, `{발행금액}`, `{공급자_입금계좌}` 등

## 8. 기존 자료 분석 결과

### 8.1 `고객정보.xlsx`
- 거래처 **22곳** (번호 1~21, 23 — 22번 누락)
- 컬럼: 번호 · 담당자 · web · **메일** · 상호 · 파일명 · `Merge status` · `Attachment name` · `Attachment`
- `Merge status`/`Attachment` 컬럼 → **기존 YAMM(Yet Another Mail Merge) 류 도구로 메일 발송**하던 것으로 추정
- 공급받는자 상세(사업자번호·대표자·주소·업태·종목)는 **없음**

### 8.2 `거래명세서 양식.xlsx`
- `sheet1 (자료입력)`: 품목·금액을 입력하는 간이 입력 영역 + "※ 부가세 별도 서식입니다." 안내
- `sheet2 (거래명세서)`: 표준 한국식 거래명세서 인쇄 레이아웃 (책번호/권/호, 보관용, 등록번호 자릿수 칸, 공급가액·세액 자릿수 칸, 품목표, 합계, 입금계좌, "이 금액을 청구함")
- 공급자 정보 확정: **예시상사 / 000-00-00000 / 홍길동 / 서울특별시 ○○구 ○○대로 000 / 서비스 / 소프트웨어 개발 및 공급 / ○○은행 000-00-000000**
- 부가세 처리: **공급가액 + 부가세(10%) = 발행금액** (부가세 별도)
