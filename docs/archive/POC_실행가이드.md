# POC 실행 가이드 — 거래명세서 자동 발송

> 목적: **테스터 거래처(테스터상사)** 로 데이터 → PDF → Drive 보관 → 메일 발송까지
> 한 바퀴를 직접 돌려보고 **정합성(출력물·동작)이 맞는지** 검증한다.
> POC 단계에서는 실제 고객이 아니라 **테스트 수신메일(test@example.com)로만** 발송된다.
>
> **출력 양식 = 기존 `거래명세서 양식.xlsx`의 파란 양식 그대로.** 스크립트는 그 양식의 `자료입력` 시트에
> 값을 채우고, 수식으로 자동 렌더링된 `세금계산서양식`(파란 거래명세서)을 PDF로 내보낸다.

소스: [gas/Code.gs](../../gas/Code.gs) · [gas/Setup.gs](../../gas/Setup.gs) · [gas/appsscript.json](../../gas/appsscript.json)

---

## 사전 준비
- 발송에 쓸 **Google 계정**(개인 Gmail 가능) 로그인 상태
- `docs/거래명세서 양식.xlsx` 파일 (기존 파란 양식)

---

## 1단계 — 양식 파일을 구글 시트로 열기
가장 간단한 방법은 **기존 양식 파일 자체를 구글 시트로 변환**해서 그 안에서 작업하는 것입니다.
1. `거래명세서 양식.xlsx` 를 **Google Drive에 업로드**
2. 그 파일을 더블클릭해서 열고 → 상단 **파일 → Google Sheets로 저장** (또는 자동 변환)
3. 변환된 구글 시트에 **`자료입력` / `세금계산서양식`** 두 탭이 보이면 성공
4. 파일 이름을 `거래명세서_자동화` 등으로 변경

> 대안) 이미 빈 시트로 작업 중이라면 → **파일 → 가져오기 → 업로드**로 `거래명세서 양식.xlsx`를 올리고
> **"새 시트 삽입"** 을 선택하면, 기존 워크북에 두 탭이 추가됩니다.

## 2단계 — Apps Script 열고 코드 붙여넣기
1. 상단 메뉴 **확장 프로그램 → Apps Script**
2. (선택) 좌측 ⚙️ **프로젝트 설정 → "에디터에 appsscript.json 표시" 체크** → 매니페스트 붙여넣기
3. 스크립트 파일 2개를 맞춘다:
   - `Code.gs` ← [gas/Code.gs](../../gas/Code.gs) 내용 전체 복사 → 붙여넣기
   - `Setup.gs` ← `+` → 스크립트 추가, 이름 `Setup` → [gas/Setup.gs](../../gas/Setup.gs) 내용 붙여넣기
4. 💾 저장

> 💡 `appsscript.json`은 안 건드려도 됩니다(첫 실행 때 권한 자동 요청).

## 3단계 — 설치 실행 (데이터 시트 생성)
1. 함수 드롭다운에서 **`setupSheets`** 선택 → **실행 ▶**
2. 최초 1회 **권한 승인**: "고급 → (안전하지 않음으로) 이동 → 허용"
   - (구글 미인증 개인 스크립트라 뜨는 정상 단계. 본인 시트/메일/드라이브만 사용)
3. 완료되면 **설정 / 거래처 / 거래내역 / 발송로그** 탭이 추가됨 (기존 `자료입력` / `세금계산서양식` 은 그대로 유지)
4. 시트로 돌아오면 상단에 **`거래명세서 자동화`** 커스텀 메뉴 생성

> ⚠️ `자료입력` / `세금계산서양식` 탭이 없으면 설치가 중단되고 안내가 뜹니다 → 1단계를 먼저 하세요.

## 4단계 — 수신 메일 확인
- `설정` 시트 **`테스트_수신메일`** = `test@example.com` (POC는 이 주소로만 발송)
- `관리자_이메일` = 본인 메일 자동 입력 (월간 발송결과 요약 수신용)

## 5단계 — POC 실행 ▶
- 메뉴 **`거래명세서 자동화 → ② POC 테스트`** 클릭 (또는 `runPOC` 실행)
- 권한 창이 한 번 더 뜨면(메일·드라이브) → 허용
- 끝나면 요약 팝업: 거래처/청구월/합계/PDF 보관 링크 표시

## 6단계 — 실전 전환 (정합성 OK일 때)
1. **`거래내역` 시트에 이번 달 실제 품목 입력** (샘플 행은 지우기)
   - 거래처ID는 `거래처` 시트의 ID(C001…) 사용 / **한 거래처당 품목 최대 4개**(양식 행 수)
2. `설정` 시트 **`발송_활성화` → `TRUE`** (이제 실제 거래처에게 발송)
3. 점검용으로 메뉴 **`이번 달 발송 실행 (전체)`** 수동 실행
4. 자동화: 메뉴 **`매월 10일 자동발송 등록`** → 매월 10일 09시 자동 발송
   - 발송 후 관리자에게 `성공 X / 실패 Y` 요약 메일
   - 해제: **`자동발송 해제`**

## 동작 요약
```
설정/거래처/거래내역 ─> [자료입력 시트에 값 주입] ─> 세금계산서양식(파란 양식, 수식 자동) ─> PDF ─┬─> Gmail 발송
                                                                                                  └─> Drive 연/월 보관 ─> 발송로그
```

## 자주 묻는 문제
| 증상 | 원인/해결 |
|------|-----------|
| 설치 시 "양식을 먼저 가져오세요" | 1단계(양식을 구글시트로) 후 다시 `setupSheets` |
| "시트 없음: 자료입력" 오류 | 양식 탭 이름이 `자료입력` / `세금계산서양식` 인지 확인 |
| 권한 화면에서 막힘 | "고급 → 이동 → 허용" 진행 |
| 메일이 안 옴 | `테스트_수신메일` 확인 / 스팸함 / 하루 100통 한도 |
| PDF 양식이 깨지거나 색이 다름 | 가져오기 시 서식 일부 변형 가능 → 알려주시면 보정 |
| 품목이 5개 이상 | 양식이 4행이라 "실패+통지" 처리. 4개 이하로 입력(필요 시 양식 행 확장 가능) |
